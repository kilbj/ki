HTTP 协议入门：浏览器背后的故事

2025-01-15 · 9 min · 网络

当你在浏览器地址栏输入一个网址并按下回车键，到页面完全展示——这短短几百毫秒里发生了什么？这篇文章带你一步步拆解。

第一步：DNS 解析

首先，浏览器需要知道 example.com 对应的 IP 地址。它依次查询：

  1. 浏览器 DNS 缓存
  2. 操作系统 DNS 缓存
  3. 本地 hosts 文件
  4. 递归查询 DNS 服务器（通常是 ISP 或 8.8.8.8）

一旦拿到 IP 地址，浏览器就可以建立 TCP 连接了。

第二步：TCP 三次握手

HTTP 建立在 TCP 之上。建立一个 TCP 连接需要三次通信：

客户端 ──SYN──→ 服务器
客户端 ←─SYN+ACK─ 服务器
客户端 ──ACK──→ 服务器

这个过程确保双方都有收发数据的能力。

第三步：TLS 握手（HTTPS）

如果是 HTTPS，TCP 连接建立之后还有 TLS 握手：

  • 客户端发送支持的加密套件列表
  • 服务器选择加密套件并发送证书
  • 客户端验证证书（CA 签名链）
  • 双方协商对称加密密钥

完成之后，所有传输的数据都是加密的。

HTTPS 不是可选项，而是必需品。Google 将 HTTPS 作为排名因素已经多年，浏览器也开始标记 HTTP 页面为"不安全"。何况还有 Let's Encrypt 提供免费证书，没有理由不用。

第四步：HTTP 请求

连接建立后，浏览器发送 HTTP 请求：

GET / HTTP/1.1
Host: example.com
Accept: text/html
User-Agent: Mozilla/5.0 ...
Connection: keep-alive

HTTP/1.1 默认支持 keep-alive，同一个 TCP 连接可以复用发送多个请求。

第五步：HTTP 响应

服务器返回响应：

HTTP/1.1 200 OK
Content-Type: text/html
Content-Length: 1234
Cache-Control: max-age=3600

<!DOCTYPE html>
<html>...</html>

第六步：浏览器渲染

浏览器解析 HTML，构建 DOM 树，同时下载 CSS 构建 CSSOM 树，两者合并成渲染树，然后经过布局（Layout）和绘制（Paint）步骤展示在屏幕上。

如果 HTML 中有 <script> 标签（不带 async 或 defer），解析会暂停，下载并执行 JavaScript 后再继续——这就是为什么脚本通常放在页面底部。

HTTP/2 和 HTTP/3

现在大多数网站已经使用 HTTP/2，它支持多路复用（一个连接同时发送多个请求）、头部压缩和服务器推送。而 HTTP/3 更进一步，基于 QUIC（基于 UDP），连接建立更快，尤其适合弱网环境。

理解 HTTP 的工作原理，能帮助你更好地调试网络问题、优化页面加载性能，也能让你在面试中脱颖而出。

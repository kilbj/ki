前端工程师的 Docker 入门实践

2024-12-05 · 7 min · 工具

Docker 不再是后端的专利。前端开发中用 Docker 做环境统一、CI/CD 部署，能省下大量"在我电脑上能跑啊"的时间。

为什么前端要用 Docker？

  • 环境一致 — 团队统一 Node 版本、系统依赖
  • 快速部署 — 多阶段构建让镜像更小，部署更快
  • 隔离 — 不同项目用不同环境，互不干扰

一个简单的 Nginx 部署

# Dockerfile
FROM node:18-alpine AS build
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
RUN npm run build

FROM nginx:alpine
COPY --from=build /app/dist /usr/share/nginx/html
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]

多阶段构建的好处是最终镜像只包含 Nginx 和构建产物，没有 node_modules 等冗余文件。

docker-compose 开发环境

用 docker-compose 可以同时启动前端、后端和数据库，一条命令拉起整个开发环境。

Docker 的学习曲线对前端来说不算陡。先掌握 Dockerfile 和 docker-compose 就够日常用了。

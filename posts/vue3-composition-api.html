Vue 3 Composition API 实战笔记

2024-12-20 · 8 min · 前端

Vue 3 的 Composition API 提供了一种全新的逻辑组织方式。本文分享从 Options API 迁移过程中的心得。

为什么选择 Composition API？

Options API 在组件逻辑复杂时，同一个功能的代码分散在 data、methods、computed 中，难以维护。Composition API 让我们按功能组织代码。

// 使用 setup 函数组织逻辑
export default {
  setup() {
    const count = ref(0);
    const double = computed(() => count.value * 2);
    function increment() { count.value++; }
    return { count, double, increment };
  }
};

ref vs reactive

ref 用于基本类型和需要重新赋值的场景，reactive 用于对象。实践中推荐优先使用 ref，它的语义更统一。

Composition API 不是要取代 Options API，而是提供更多的灵活性。在简单组件中继续使用 Options API 完全没问题。

自定义 Hook 复用

将可复用的逻辑提取为以 use 开头的函数，这是 Composition API 最强大的特性之一。
